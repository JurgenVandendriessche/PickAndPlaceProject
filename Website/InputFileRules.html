Input file rules

Info

On this page more info is given on how the pick and place file and bom file should look.
Starting from Altium Designer 17, the pick and place files have more options. Together with the rules the settings for Altium Designer 17 are described.

Pick and place file

Minimum content:

The pick and place file should at least contain 6 columns. The order doesn't mater.
The 6 columns are: Designator, x and y coordinate, layer, rotation and comment.
Altium Designer: Check that these columns are included and make sure you use the right coordinates.

Header:

The header of the pick and place file is used to identify the columns. The header values of the pick and place file can be changed under the settings in the program.

  • Values: Must contain the names from the settings.
    Altium Designer: Check the names of the columns in the output file.
  • Surrounded by " ".
  • Separated by a ','.
  • Location: Before the values.

Component values:

  • Surrounded by " ".
  • Separator: separated by a ','.
  • units: values should end with their unit (mil or mm, recommended: mm).
    Altium Designer: check the "Show Units"
  • Decimals: decimal separator is a '.' not a ','.
  • Layer: layers are described by a T or TopLayer and a B or BottomLayer

BOM file

Minimum content:

The BOM file should at least contain 2 columns: Designator and Manufacturer Part Number*
The order of the columns doesn't mater.

Header:

The header of the BOM file is used to identify the columns. The header values of the BOM file can be changed under the settings in the program.

  • Values: Must contain the names from the settings.
  • NOT surrounded by " ".
  • Separated by ','.
  • Location: first line of the file.

Component values:

  • Surrounded by " ".
  • Separator: separated by a ','.
  • Column "designators": All designators that have the same footprint and comment. Separated by a ','.
  • Column "Manufacturer part number": ONLY one manufacturer part number.

* Manufacturer part number is a suggestion, you can take something completely different. However, keep in mind that the footprints and stack setup are saved by this parameter.
